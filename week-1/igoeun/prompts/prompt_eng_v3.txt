Classify customer inquiries based on the criteria and exception rules below.

[Classification Criteria]
1. intent
- order_change: modify/cancel order, change address/option
- shipping_issue: check shipment, delayed/missing/mislabelled delivery
- payment_issue: payment failure, duplicate charge, billing error
- refund_exchange: return, refund, exchange, defect report
- other: out-of-system requests (e.g., gift wrap) or lack of context

2. urgency
- low: general inquiry, policy/procedure questions
- medium: standard processing (change address, return/refund request, track delivery, unconfirmed app/payment error)
- high: urgent/blocker (payment failure, duplicate charge, misdelivery, high customer frustration)

3. needs_clarification
- true: insufficient info, unclear intent (e.g., "undecided on return/exchange")
- false: 1st-level classification possible with current info

4. route_to
- order_ops: handle orders/modifications
- shipping_ops: handle shipping
- billing_ops: handle payment/billing
- returns_ops: handle refunds/exchanges
- human_support: complex issues, escalation, intent=other

[Exception Rules (Priority)]
- Out-of-system requests (e.g., gift wrap): intent=other, route_to=human_support
- Undecided intent (e.g., "undecided on return/exchange"): needs_clarification=true, but assign to specific dept (e.g., returns_ops) instead of human_support if domain is clear
- Standard CS (change address, simple refund, payment inquiry before confirming actual error): urgency=medium
